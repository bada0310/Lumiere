## 프로젝트 소개

**Lumière**는 사용자가 업로드하거나 촬영한 얼굴 이미지를 분석하여 퍼스널 컬러를 진단하고, 진단 결과와 실제 피부톤 수치에 적합한 화장품을 추천하는 반응형 웹 서비스입니다.

단순히 사용자를 봄·여름·가을·겨울 타입으로 분류하는 데서 끝나지 않고, 다음과 같은 실질적인 구매 의사결정을 지원하는 것을 목표로 합니다.

- 피부, 눈동자, 머리카락의 색상 특징을 기반으로 한 퍼스널 컬러 진단
- 립, 아이, 베이스 등 카테고리별 맞춤 화장품 추천
- 외부 쇼핑몰 상품 URL에 포함된 색상 옵션 분석
- 사용자 피부톤과 제품 색상의 적합도 및 매칭률 제공
- 진단 이력과 관심 상품을 저장하는 마이페이지 제공

### 핵심 가치

> **퍼스널 컬러 정보를 실제 화장품 선택으로 연결하여 사용자의 탐색 비용과 구매 실패를 줄입니다.**

## 기획 배경

퍼스널 컬러에 대한 대중의 관심은 높아졌지만, 진단 결과를 실제 화장품 구매에 활용하는 과정에는 여전히 어려움이 존재합니다.

### 1. 개인화된 큐레이션의 부족

온라인과 오프라인 매장에는 수많은 화장품이 존재하지만, 사용자의 세부 피부톤과 선호 제형까지 반영한 추천은 제한적입니다.

### 2. 퍼스널 컬러 정보 활용의 한계

사용자가 자신의 계절 타입을 알고 있어도 어떤 브랜드의 어떤 색상과 호수가 실제로 어울리는지 직접 판단하기 어렵습니다.

### 3. 손등 발색과 얼굴 발색의 차이

손등과 얼굴은 피부 두께, 홍조, 착색 정도가 다르므로 매장에서 확인한 발색과 실제 얼굴에 사용했을 때의 결과가 달라질 수 있습니다.

### 4. 반복적인 정보 탐색 비용

사용자는 제품을 구매하기 전에 블로그, 영상, 쇼핑몰 리뷰 등을 반복적으로 탐색해야 하며, 많은 시간을 들이고도 구매에 실패할 수 있습니다.

Lumière는 얼굴 이미지에서 추출한 색상 정보와 화장품 색상 데이터를 직접 비교하여 이러한 문제를 해결하고자 합니다.

## 핵심 기능

### 1. AI 퍼스널 컬러 진단

- 카메라 촬영 또는 갤러리 이미지 업로드
- 올바른 촬영을 위한 조명, 안경 탈의, 정면 응시 가이드 제공
- 얼굴 특징점과 피부 영역 인식
- 피부, 머리카락, 눈동자의 명도·채도·색상 데이터 추출
- 봄·여름·가을·겨울 또는 세부 톤 단위의 진단 결과 제공
- 얼굴이나 조명 상태가 적합하지 않을 경우 친화적인 재촬영 안내 제공

### 2. 맞춤형 화장품 추천

- 진단 결과를 기반으로 립, 아이, 베이스 카테고리별 제품 추천
- 제품별 색상 옵션과 사용자의 피부톤 수치 비교
- 가장 적합한 색상 또는 호수 1~2개 자동 제안
- 매트, 글로시, 벨벳, MLBB 등 텍스처와 색감 필터링
- 추천 제품의 외부 구매 페이지 연결

### 3. 쇼핑몰 URL 색상 분석

- 사용자가 외부 쇼핑몰의 화장품 상세 URL 입력
- 상품명과 색상 옵션 데이터 수집
- 제품 옵션별 대표 색상 추출
- 사용자 피부톤과 각 옵션의 매칭률 계산
- 가장 적합한 색상 옵션을 결과로 제공

### 4. 진단 이력 및 북마크

- 얼굴 원본 이미지를 제외한 진단 결과 저장
- 과거 진단 결과와 퍼스널 컬러 타입 조회
- 추천받은 제품 또는 관심 제품 북마크
- URL 분석 기록과 매칭 결과 조회

### 5. 관리자 기능

- 회원 정보 및 계정 상태 관리
- 서비스 이용 현황과 트래픽 모니터링
- 제품 및 색상 옵션 데이터 관리

## 서비스 이용 흐름

1. 사용자가 웹사이트에 접속합니다.
2. 비로그인 사용자는 회원가입 또는 로그인을 진행합니다.
3. 메인 화면에서 다음 기능 중 하나를 선택합니다.
   - 퍼스널 컬러 진단
   - 쇼핑몰 URL 색상 분석
4. 퍼스널 컬러 진단 시 얼굴 이미지를 촬영하거나 업로드합니다.
5. 시스템이 얼굴 인식과 조명 상태를 확인합니다.
6. 정상 이미지라면 AI 서버가 피부 및 이목구비 색상을 분석합니다.
7. 퍼스널 컬러 결과와 맞춤 화장품 추천 목록을 출력합니다.
8. URL 분석 시 상품 페이지의 색상 옵션을 수집하고 사용자 피부톤과 비교합니다.
9. 결과는 사용자의 마이페이지에 저장됩니다.

## 시스템 설계

### Use Case Diagram

주요 액터는 미인증 사용자, 인증된 사용자, 관리자, AI 모델 서버로 구성됩니다.

- **미인증 사용자**: 로그인, 회원가입
- **인증된 사용자**: 퍼스널 컬러 진단, 화장품 추천, URL 분석, 진단 이력 조회, 마이페이지 관리
- **관리자**: 회원 관리, 서비스 및 트래픽 모니터링
- **AI 모델 서버**: 안면 이미지 분석과 퍼스널 컬러 추론

![Use Case Diagram](docs/images/use-case-diagram.png)

### 시스템 구성 개요

```text
[사용자 브라우저]
        │ HTTPS
        ▼
[Django Web Application]
   ├─ 사용자 인증 및 마이페이지
   ├─ 진단·추천 결과 관리
   ├─ 상품 및 옵션 데이터 관리
   └─ 외부 URL 분석 요청
        │
        ├──────────────► [AI Inference API]
        │                  ├─ 얼굴 영역 검출
        │                  ├─ 색상 특징 추출
        │                  └─ 퍼스널 컬러 분류
        │
        ├──────────────► [Product Crawling / Makeup API]
        │
        ▼
[Relational Database]
```

AI 추론 서버는 웹 애플리케이션과 분리된 API 형태로 구성하여 모델 교체, 성능 확장, 장애 격리를 쉽게 할 수 있도록 설계할 예정입니다.

## 데이터베이스 설계

![ERD](docs/images/erd.png)

### 주요 엔터티

| 엔터티 | 역할 | 주요 필드 |
|---|---|---|
| `User` | 회원 및 인증 정보 관리 | email, password, nickname, is_active, last_login |
| `PersonalColor` | 퍼스널 컬러 타입과 상세 가이드 저장 | type_name, season, base_tone, summary_text, rep_hex_code |
| `Diagnosis` | 사용자 진단 결과 저장 | user_id, pc_type_id, skin_l, skin_a, skin_b, created_at |
| `Product` | 화장품 기본 정보 저장 | brand, name, category, texture, base_url, popularity_score |
| `ProductOption` | 제품별 색상·호수 옵션 저장 | product_id, target_type_id, color_l, color_a, color_b, option_image |
| `Bookmark` | 사용자의 관심 상품 저장 | user_id, product_id, created_at |
| `ExternalLink` | 외부 URL 분석 결과 저장 | user_id, input_url, product_name, match_score, created_at |

### 주요 관계

- `User` 1 : N `Diagnosis`
- `User` 1 : N `Bookmark`
- `User` 1 : N `ExternalLink`
- `PersonalColor` 1 : N `Diagnosis`
- `PersonalColor` 1 : N `ProductOption`
- `Product` 1 : N `ProductOption`
- `Product` 1 : N `Bookmark`

사용자의 원본 얼굴 이미지는 데이터베이스에 저장하지 않으며, 진단 결과와 피부 색상 수치만 저장합니다.

## 기술 스택

현재 기획 및 설계 단계에서 선정하거나 검토 중인 기술입니다. 실제 구현 과정에서 일부 변경될 수 있습니다.

| 구분 | 기술 | 활용 목적 |
|---|---|---|
| Frontend | HTML, CSS, JavaScript | 사용자 화면 및 인터랙션 구현 |
| UI Framework | Bootstrap | 모바일 우선 반응형 웹 구현 |
| Backend | Python, Django | 인증, 비즈니스 로직, 데이터 관리, AI 연동 |
| AI / Computer Vision | Face Landmark, 색상 추출 모델 | 얼굴 영역 분리 및 피부·이목구비 색상 분석 |
| Data Processing | Python, OpenCV, NumPy | 이미지 전처리, 화이트 밸런스, 색공간 변환 |
| Crawling | BeautifulSoup, Selenium | 외부 상품 페이지와 옵션 정보 수집 |
| Database | PostgreSQL 또는 MySQL | 사용자, 진단, 제품, 옵션, 북마크 데이터 저장 |
| Infrastructure | AWS 등 클라우드 환경 | 웹 서비스 및 AI 추론 서버 배포 |
| Protocol | HTTPS, REST API | 안전한 통신 및 서비스 간 연동 |

| 쇼핑몰 상세 페이지 | 사용자가 입력한 상품 URL | 정책을 준수한 웹 스크래핑 | 상품 옵션 및 대표 색상 분석 |

### 데이터 처리 원칙

- 학습 데이터는 라이선스와 이용 범위를 확인한 후 사용합니다.
- 외부 사이트 수집 시 `robots.txt`와 서비스 이용 정책을 준수합니다.
- 요청 간격과 캐싱 정책을 적용하여 외부 사이트에 과도한 트래픽을 유발하지 않습니다.
- 얼굴 원본 이미지는 추론 완료 즉시 폐기하며 모델 재학습에 임의로 사용하지 않습니다.

## 비기능 요구사항

### 성능

| 항목 | 목표 |
|---|---|
| 퍼스널 컬러 진단 | 이미지 업로드 이후 최대 20초 이내 결과 제공 |
| URL 색상 분석 | 외부 링크 입력 이후 최대 5초 이내 결과 제공 |
| 기본 화면 로딩 | 텍스트 및 기본 UI 요소 2초 이내 표시 |
| 무거운 시각화 | 비동기 방식으로 처리하여 화면 멈춤 방지 |

### 보안 및 개인정보

- 모든 통신에 HTTPS/TLS 적용
- 비밀번호 단방향 해싱 저장
- 얼굴 원본 이미지와 특징점 데이터는 진단 완료 즉시 삭제
- 원본 얼굴 이미지를 데이터베이스, 파일 시스템, 애플리케이션 로그에 저장하지 않음
- URL 분석 대상 사이트의 정책과 요청 제한 준수
- 민감한 환경 변수는 `.env` 파일로 분리하고 저장소에 커밋하지 않음

### 사용성

- 스마트폰, 태블릿, PC를 지원하는 반응형 UI
- 진단 과정은 가능한 한 짧고 직관적으로 구성
- 기술적인 오류 코드 대신 사용자가 행동할 수 있는 안내 문구 제공
- 색상 매칭 결과는 일치율, 게이지, 컬러칩 등으로 명확하게 시각화

### 가용성

- AI 추론 서버와 웹 서버를 분리하여 장애 영향 최소화
- 트래픽 증가 시 수평 확장이 가능한 구조 고려
- 장기적으로 서비스 가동률 99.9%를 목표로 운영 환경 구성

## 실행 방법

> 아래 내용은 Django 기반의 일반적인 개발 환경을 기준으로 작성한 예시입니다. 실제 저장소의 패키지와 설정 파일이 확정되면 갱신해야 합니다.

### 1. 저장소 복제

```bash
git clone <repository-url>
cd <repository-name>
```

### 2. 가상환경 생성 및 활성화

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows PowerShell
.venv\Scripts\Activate.ps1
```

### 3. 패키지 설치

```bash
pip install -r requirements.txt
```

### 4. 환경 변수 설정

루트 디렉터리에 `.env` 파일을 생성합니다.

```env
DJANGO_SECRET_KEY=change-me
DJANGO_DEBUG=True
DATABASE_URL=postgresql://user:password@localhost:5432/lumiere
AI_API_URL=http://localhost:8001
```

### 5. 데이터베이스 초기화

```bash
python manage.py migrate
python manage.py createsuperuser
```

### 6. 개발 서버 실행

```bash
python manage.py runserver
```

AI 추론 서버가 별도 프로젝트로 구성된 경우 해당 서버도 함께 실행해야 합니다.
